## 해시 테이블이 뭔가요?
### 정제한 답변
해시 테이블은 해시 함수를 사용하여 데이터를 배열의 인덱스로 변환한 후 해당 인덱스 위치에 데이터를 저장하는 자료구조입니다. 
이 방식 덕분에 삽입, 삭제, 조회 작업을 평균 O(1)의 시간 복잡도로 수행할 수 있습니다. 
해시테이블이 없었다면, 테이블의 모든 값을 직접 찾아야 해서 최악의 경우 O(N)이 소요되는데요, 이를 해결하기 위해 고안된 방법입니다. 
여기서 해시 값이 같은 경우가 있을 수 있는데, 이를 해시 충돌이라고 합니다.
해시 충돌을 해결하는 방법은 체이닝과 개방 주소법이 있습니다.
체이닝은 배열의 원소를 연결리스트로 두어 인덱스 값이 같을 때 연결리스트의 데이터를 순회하며 값이 같은 요소를 찾습니다. 자바의 HashMap이 체이닝을 통해 동작하며, Java 8부터 한 버킷에 8개를 초과한 원소가 쌓이면 연결 리스트를 레드블랙 트리로 전환합니다. 이를 통해 시간 복잡도를 O(N)에서 O(logN)까지 줄일 수 있습니다. 
개방 주소법은 충돌이 나면 다른 빈 버킷으로 이동합니다. 선형 탐사와 이중 해싱이 대표적입니다.
