## HTTP에 대해서 설명해 주세요.
### 정제한 답변
HTTP가 없었을 때는 요청/응답에 사용되는 프로토콜이 통일되지 않아 통신에 불편함이 있었다. HTTP는 이런 불편함을 해소하고자 고안되었다.
HTTP는 HTTP Method, 응답 코드 등을 바탕으로 메시지의 정보를 표현한다.
GET, POST, DELETE, PUT, HEAD 등이 있는데, 모두 “HTTP 메시지의 행동"을 표현한다. 
응답 코드는 200, 300, 400, 500번대가 있으며 각각 200번대는 요청 응답이 성공됨, 300번대는 리다이렉트, 400번대는 클라이언트 잘못, 500번대는 서버 에러로 구분지어 뒤에 숫자로 규격화된 정보를 제공한다.
HTTP는 결국 인터넷에서 통신을 원활하게 하기 위함이 출발 목적이었고, HTTP 2와 3 모두 비슷한 목적을 가진다고 볼 수 있겠다.
